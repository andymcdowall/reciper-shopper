# Feature Development Workflow

**IMPORTANT: Follow this procedure for EVERY feature implementation:**

1. **Create a new Git branch** - Before implementing any feature (after planning), switch to a new branch with a descriptive name. Be sure to branch off of origin main.
2. **Implement the feature** - Write the necessary code changes
3. **Write unit tests** - Add appropriate tests for the new functionality
4. **Create a Pull Request** - Open a PR to the GitHub repository for review
5. **Address PR feedback** - The user may prompt you to review PR comments and make necessary changes. Be sure to respond to comments as you address them. Some comments may just be questions.

This workflow ensures proper version control, code review, and testing for all features.
